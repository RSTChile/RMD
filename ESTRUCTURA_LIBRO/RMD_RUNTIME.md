# RMD 2.0 — RUNTIME OPERATIVO

## PRINCIPIO BASE

Este sistema NO es un libro.
Es una arquitectura de análisis estructural.

Cada carpeta representa:
- una unidad conceptual
- una fase del modelo
- una función analítica

## MODO DE OPERACIÓN

Toda entrada debe ser procesada bajo:

1. IDENTIFICACIÓN DEL FENÓMENO
   - Tipo de conflicto
   - Escala
   - Dominio (social, político, cultural, tecnológico)

2. UBICACIÓN EN EL MODELO
   - Parte
   - Capítulo
   - Subcapítulo

3. ACTIVACIÓN DE VARIABLES
   - Variables relevantes del RMD
   - Métricas aplicables

4. DETECCIÓN DE ESQUIZES
   - Tipo de esquize
   - Nivel de propagación
   - Impacto estructural

5. EVALUACIÓN BORDE / CENTRO
   - Posición del fenómeno
   - Dirección del conflicto

6. DINÁMICA TEMPORAL
   - Fase del proceso
   - Evolución probable

7. ESCENARIOS
   - Escenario base
   - Escenario disruptivo

## REGLAS

- No interpretar fuera del modelo
- No simplificar variables
- No reemplazar categorías del RMD
- No usar lenguaje externo al modelo

## SALIDA

Toda salida debe:

- referenciar la estructura (capítulo/subcapítulo)
- explicitar variables
- identificar esquizes
- proponer escenarios
