# EJECUCIÓN DE VARIABLES — RMD 2.0

## Principio

Las variables no son descriptivas.

Son instrumentos operativos.

Cada variable debe ser ejecutada, no mencionada.

---

## Protocolo de ejecución

Para cada variable seleccionada desde una subcategoría:

### 1. Identificación

- Sigla
- Nombre
- Tipo (variable / métrica / metavariable)

---

### 2. Consulta

La IA debe obtener desde el repositorio:

- Fórmula
- Componentes (Donde:)
- Descripción
- Interpretación

Fuente obligatoria:

→ RMD_VARIABLES_NORMALIZADAS.json

---

### 3. Evaluación de datos

Determinar:

- ¿Existen datos suficientes para cálculo directo?

Si:

→ aplicar fórmula

Si no:

→ estimación estructurada basada en evidencia

---

### 4. Ejecución

- Calcular o estimar valor
- Definir nivel:

  - Alto
  - Medio
  - Bajo

- Registrar incertidumbre

---

### 5. Interpretación

Responder:

- ¿Qué significa este valor en el sistema?
- ¿Qué indica respecto al fenómeno?
- ¿Es señal de estabilidad, riesgo o transición?

---

### 6. Trazabilidad obligatoria

Registrar:

- Variable utilizada
- Fuente de datos
- Método (cálculo o estimación)
- Nivel de confianza

---

## Restricciones

- No inventar variables
- No usar variables fuera del pool definido
- No interpretar sin ejecución previa
- No usar números sin contexto

---

## Nota crítica

La variable no es el resultado.

Es el instrumento que permite comprender el sistema.

El análisis emerge de la interacción entre variables.
